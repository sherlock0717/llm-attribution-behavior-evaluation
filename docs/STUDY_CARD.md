# 研究说明

## 研究对象

本项目观察语言模型如何评价一个行动者。材料控制行动者身份和决策过程表述，模型随后对行动控制、理由响应、选择自主性、体验性、感知智能和责任等内容评分。

研究问题是：

> 当最终决定相同或接近时，理由比较、反思修正和身份标签会怎样改变模型的归因判断？

## 分析单位

一条记录对应模型对一份条件化材料给出的评分响应。记录中的“参与者编号”用于运行和数据组织，不代表现实中的人类被试。

## 输入设计

当前任务交叉组合：

- 八个决策情境；
- 六种决策过程表述；
- AI 决策者与人类决策者两种身份标签；
- 多个归因题项及相应评分范围。

过程条件覆盖直接给出选择、增加背景长度、列出候选方案、比较理由以及根据反馈修正行动等文本线索。

## 输出指标

当前题项组织为以下几类：

- **能动性**：行动控制、目标维持、理由响应和行为修正；
- **选择自主性与自由意志归因**：选择所有权、替代可能性和外部推动；
- **体验性**：模型是否把行动者看作具有感受能力；
- **感知智能**：理解、逻辑和判断质量；
- **责任相关判断**：结果责任、道德评价和过程解释责任；
- **过程检查**：材料是否呈现候选方案、理由比较与反馈修正。

题项根据既有理论和测量构念进行情境化编写。原研究的信度、效度和因素结构不能直接转移到当前题项，详细来源见 [`research_and_measurement_sources.md`](research_and_measurement_sources.md) 与 [`scale_source_mapping.md`](scale_source_mapping.md)。

## 当前数据

公开结果来自 DeepSeek API 对当前材料的评分输出。仓库保留聚合数据、分析代码和统计产物；早期运行没有保存完整的模型服务端快照、逐请求 Prompt 快照、token、费用和精确时间信息，因此无法逐请求还原当时运行。

确定性 `mock` 数据用于验证任务配置、解析、校验、计分、恢复和产物生成，不进入研究结论。

## 当前结果的使用方式

现有结果适合回答：

- 模型是否会对不同过程表述给出系统性不同的评分；
- 哪些归因维度对身份标签更敏感；
- 某些差异能否由文本长度或感知智能解释；
- 哪些情境和条件容易产生不稳定或相互矛盾的判断。

当前结果不用于判断模型是否具有自由意志，也不代表人类总体心理规律。

## 已知方法问题

后续研究需要优先处理：

1. 部分过程条件同时改变理由质量、信息密度、道德合理性和修正能力；
2. 能动性、自主性和自由意志题项之间存在语义重叠；
3. 同一模型对相似材料的重复调用不等同于相互独立的人类样本；
4. 现有数据来自单一模型，尚未检验跨模型和跨 Prompt 改写的稳健性；
5. 早期运行的逐请求来源信息不完整。

因此，当前公开结果以描述性比较和关联性诊断为主。新的模型运行应保存完整配置、模型标识、Prompt、原始响应、错误记录和运行清单，并使用按场景或改写版本聚类的统计方法。

## 复现范围

仓库当前可以复现：

- 依赖安装与测试；
- 任务配置加载；
- 确定性模拟运行；
- 响应解析、校验和计分；
- 分析产物与展示页数据生成；
- 运行文件的哈希和清单记录。

早期 DeepSeek 输出可用于复核现有聚合结果，但不能逐请求复现。
